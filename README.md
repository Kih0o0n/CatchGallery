# 캐치갤러리

그림을 게시하고, 다른 사용자가 나중에 정답을 맞히는 모바일 우선 비동기 그림 퀴즈입니다.

## v1.4.0 그림판 색상과 반짝이 색연필

- 그림판 색상이 16색으로 새롭게 바뀌었어요.
- 연분홍, 연주황, 연두, 연보라 색상이 추가됐어요.
- 반짝이는 금색과 은색 색연필이 추가됐어요.
- 금색과 은색도 끊김 없이 부드럽게 그려져요.
- 스마트폰과 태블릿에서도 색상을 편하게 고를 수 있도록 배치를 다듬었어요.

## v1.3.0 기기 호환성과 안정성 강화

- 스마트폰·태블릿·PC 화면 크기에 맞춰 전시장, 관리 화면과 그림판 배치를 개선했습니다. 모바일 세로 화면에서는 그림판 공간을 넓히고 safe area와 짧은 화면에 대응했습니다.
- iPad와 인앱 브라우저에서 발생할 수 있는 끊김, 긴 연결선과 입력 고착을 방어하고 Apple Pencil과 손가락 입력 전환을 안정화했습니다. 점 찍기, 두 손가락 확대·이동, 전체 지우기 확인과 되돌리기도 지원합니다.
- 빠른 화면 전환 중 오래된 결과가 새 화면을 덮지 않도록 보강하고, 이미지 지연 로딩·동시 로딩 제한·제한형 사용자별 캐시로 화면 안정성과 성능을 개선했습니다. Firebase 목록 재조회와 의견 게시판·랭킹 로딩도 줄였습니다.
- 로그인·회원가입·자동 로그인 중 프로필 준비 흐름을 안정화하고, 중복되거나 오래된 인증 작업과 실패한 가입 데이터가 안전하게 정리되도록 개선했습니다.
- 저장형 XSS와 안전하지 않은 데이터 ID를 방어하고 이미지 형식·크기 검증을 강화했습니다. 그림 게시·수정 저장의 안정성을 높이고 중단된 임시 그림 정리와 기존 그림 변환 흐름을 보강했습니다.
- 제시어를 기존 403개에 신규 596개를 더한 총 999개로 확장했습니다. 유명인, 게임, 게임 캐릭터, 만화·애니 캐릭터, 유명 캐릭터·마스코트, 스타크래프트 1 유닛 카테고리를 추가했으며 카테고리가 다른 동명 제시어도 각각 유지합니다.

## 실행하기

1. Firebase Console의 Authentication에서 **이메일/비밀번호** 로그인 제공업체를 활성화합니다.
2. Authentication의 승인된 도메인에 `kih0o0n.github.io`를 추가합니다.
3. Realtime Database 규칙에 `database.rules.json` 내용을 배포합니다.
4. 정적 웹 서버로 프로젝트 폴더를 실행합니다. 예: `npx serve .`

별도의 빌드 과정은 없습니다. Firebase SDK는 공식 CDN에서 불러옵니다.

## 관리자 등록

1. 앱에서 닉네임 `admin`과 안전한 비밀번호로 회원가입합니다.
2. Authentication에서 해당 계정의 UID를 확인합니다.
3. Realtime Database의 `admins/{uid}` 값을 `true`로 저장합니다.

관리자 비밀번호는 코드, README, Database Rules 어디에도 적지 않고 Firebase Authentication에서만 검증합니다.

## 보안 안내

Firebase Web Config는 브라우저 앱에서 사용하는 공개 가능한 프로젝트 식별값입니다. Web Config를 숨기는 대신 `database.rules.json`의 Rules를 반드시 배포하고, 운영 환경에서는 Firebase App Check도 함께 활성화해야 합니다.

게임 점수는 사용자가 `users/{uid}/score`를 직접 수정하지 못하도록 잠겨 있습니다. 정답이 확정된 그림의 보상만 그림별 1회성 `scoreClaims`로 기록되며, 새 기록에는 `drawer` 또는 `solver` 유형이 함께 저장됩니다. 랭킹과 화면 점수는 이 기록을 합산해 계산합니다. 그림과 의견의 좋아요·싫어요 수도 사용자가 수정 가능한 카운터가 아니라 반응 원본에서 계산합니다.
